# Iconography

OpenScrobbler currently uses official ListenBrainz logo variants from the
MetaBrainz Design System for its app icon and in-app image assets. The assets
are copied into the Xcode asset catalog as deterministic PNG renders generated
from the local design-system checkout.

The reference design system lives at:

- https://github.com/metabrainz/design-system
- https://metabrainz.github.io/design-system

## Source Assets

The generator reads from:

- `.references/metabrainz-design-system/brand/logos/ListenBrainz/PNG/ListenBrainz_logo_square.png`
- `.references/metabrainz-design-system/brand/logos/ListenBrainz/PNG/ListenBrainz_logo_no_text.png`
- `.references/metabrainz-design-system/brand/logos/ListenBrainz/PNG/ListenBrainz_logo_short_vertical.png`
- `.references/metabrainz-design-system/brand/logos/ListenBrainz/monochrome/ListenBrainz_logo_icon_bw.png`

Clone or update the reference checkout before regenerating:

```bash
git clone https://github.com/metabrainz/design-system.git .references/metabrainz-design-system
```

## Assets

- `AppIcon.appiconset`: macOS and iOS app icon sizes.
- `ListenPulse.imageset`: official ListenBrainz no-text logo.
- `OpenGraph.imageset`: official ListenBrainz square logo.
- `LibraryScan.imageset`: official ListenBrainz no-text logo.
- `DiscoveryRadio.imageset`: official ListenBrainz short vertical logo.
- `MenuBarScrobbler.imageset`: official monochrome ListenBrainz icon rendered as a macOS template asset.

## Generation

The asset family is generated by:

```bash
python3 tools/generate_iconography.py
```

Regenerate icons after updating the MetaBrainz design-system checkout or app
icon sizing. The script writes deterministic PNGs and asset catalog metadata.

## Brand Boundaries

These files are MetaBrainz/ListenBrainz brand assets. Keep the generated renders
unaltered except for required scaling, centering, and iOS opaque-background
composition. Follow MetaBrainz logo guidance: do not stretch, recolor, add
effects, reduce opacity, or place the logo on a busy background.
